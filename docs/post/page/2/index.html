R CMD check tells me 'no visible binding for global variable'

Today is definitely a lucky day! Here is the solution to avoid the NOTE ‘no visible binding for global variable’ when running R CMD check. Basically just add to the DESCRIPTION file of your package the following line (for instance after defining the license) LazyData: yes and add a line in the routine which calls the data.frame e.g. DATA P <- DATA[,"P"] E <- DATA[,"E"] In my case, that made the package checker happy! »
